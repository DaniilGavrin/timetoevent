# Руководство пользователя TimeToEvent

> Как пользоваться приложением: от первого запуска до P2P-синхронизации.

---

## 🎯 Что это за приложение?

**TimeToEvent** — локальный трекер событий и таймеров. Отвечает на два вопроса:

- **Сколько времени до события?** (режим *countdown*)
- **Сколько прошло с события?** (режим *countup*)

Все данные хранятся **локально** на твоём устройстве. Синхронизация между устройствами — напрямую через локальную сеть (WiFi), с end-to-end шифрованием.

**Без серверов. Без интернета. Без регистрации. Без телеметрии.**

### Примеры использования

- 🎯 Таймер до дедлайна проекта
- 🚭 Счётчик «сколько я не курю»
- ✈️ Обратный отсчёт до отпуска
- 🎂 Дни рождения близких
- 💪 Трекер привычек
- 📊 Синхронизация между телефоном и ПК в одной WiFi-сети

## 🚀 Первый запуск

### Desktop (Windows / Linux)

1. Запусти приложение любым способом:
   - **В режиме разработки:** `npm run tauri dev`
   - **Production-бинарник:** установи `.msi` (Windows) или `.deb`/`.AppImage` (Linux)

2. При первом запуске увидишь **Splash Screen** с анимацией (~1.5 секунды).

3. Откроется главный экран — пока он пустой, потому что событий ещё нет.

### Android

1. Установи APK на устройство.
2. При первом запуске приложение запросит разрешение на **показ уведомлений** — это нужно для напоминаний.
3. Главный экран — аналогично desktop-версии.

### Где хранятся данные?

Все данные лежат в SQLite-файле:

| ОС | Путь |
|----|------|
| **Windows** | `%APPDATA%\rs.bytewizard.timetoevent\timetoevent.db` |
| **Linux** | `~/.local/share/timetoevent/timetoevent.db` |
| **Android** | Внутреннее хранилище приложения |

## 📝 Работа с событиями

> ⚠️ Этот раздел описывает функциональность, которая появится в **v0.2.0**.

### Создание события

1. Нажми кнопку **«+»** (или **«Создать событие»**) на главном экране.
2. Заполни форму:
   - **Название** — обязательное поле (например, «Отпуск в Турции»)
   - **Описание** — опционально
   - **Тип:**
     - *Countdown* — обратный отсчёт до события
     - *Countup* — прямой отсчёт с момента события
   - **Дата и время** — когда произойдёт событие
   - **Категория** — Работа / Личное / Здоровье / Привычки / Другое
   - **Цвет** — опционально, для визуального кодирования
3. Нажми **«Сохранить»**.

Событие появится в общем списке, отсортированном по дате.

### Просмотр и редактирование

- **Тап/клик по карточке** события — открывает детальный экран.
- На детальном экране можно:
  - Увидеть точный таймер (дни, часы, минуты, секунды)
  - Отредактировать событие (кнопка «✏️»)
  - Удалить событие (кнопка «🗑️» с подтверждением)
  - Добавить в избранное (кнопка «⭐»)

### Удаление

- При удалении появится **диалог подтверждения** — чтобы не удалить случайно.
- Удалённые события попадают в архив (в будущих версиях).

## ⏱ Таймеры

### Режим Countdown (обратный отсчёт)

Показывает, сколько времени осталось до события.

**Формат:** `Xд Yч Zм` (дни, часы, минуты)

Примеры:
- `42д 5ч 13м` — до события 42 дня, 5 часов, 13 минут
- `0д 2ч 45м` — менее суток
- `-3д` — событие прошло 3 дня назад (станет красным)

### Режим Countup (прямой отсчёт)

Показывает, сколько времени прошло с события.

**Формат:** `Xд Yч Zм` с припиской *«назад»*

Примеры:
- `127д 8ч 33м назад` — прошло 127 дней
- Полезно для трекеров привычек: «Не курю: 45д 12ч»

### Обновление в реальном времени

Таймеры обновляются **каждую секунду** автоматически — не нужно перезагружать страницу.

## 🔔 Уведомления и напоминания

### Настройка напоминания

Для каждого события можно настроить **несколько напоминаний**:

- За 5 минут
- За 1 час
- За 1 день
- За 1 неделю
- Произвольное время

### Как это работает

- Напоминания работают **пока приложение запущено** (через tokio::spawn в Rust).
- При следующем запуске приложение покажет **все пропущенные уведомления**.
- В будущих версиях — через системный планировщик (AlarmManager на Android, Task Scheduler на Windows).

### Пропущенные напоминания

Если приложение было закрыто в момент напоминания — при следующем запуске оно покажет **все пропущенные уведомления** с пометкой «(пропущено)».

## 🌐 P2P-синхронизация

> ⚠️ Этот раздел описывает функциональность **v0.3.0 – v0.4.0**.

### Как это работает

1. **Оба устройства** должны быть в **одной WiFi-сети**.
2. Приложение **автоматически обнаруживает** другие устройства с TimeToEvent через mDNS.
3. При первом обнаружении нужно **сопряжение** (pairing).

### Сопряжение устройств

1. На устройстве A нажми **«Добавить устройство»** → появится **QR-код** и **6-значный код**.
2. На устройстве B отсканируй QR-код (или введи код вручную).
3. Оба устройства покажут **6-значный код** — **визуально сравни** их.
4. Если коды совпадают — подтверди сопряжение.

> 🔒 Коды защищены **HMAC** + **ECDH (X25519)**. Даже если кто-то перехватит трафик — без визуального подтверждения кодов он не сможет подключиться.

### Что синхронизируется

- ✅ События (создание, редактирование, удаление)
- ✅ Напоминания
- ✅ Избранные
- ✅ Категории

### Конфликты

Если одно и то же событие изменили на двух устройствах одновременно — применяется правило **last-write-wins** (побеждает более позднее изменение). История конфликтов сохраняется в логе.

### Автопереподключение

Если соединение разорвалось (ушёл из WiFi, перезагрузил роутер) — приложение **автоматически переподключится** с exponential backoff (1с → 2с → 4с → ... → 30с).

## 🔒 Безопасность и приватность

### Что хранится локально

- Все события, напоминания, настройки
- Список доверенных устройств
- Ключи сессий (в памяти, не на диске)

### Что НЕ уходит в интернет

- ❌ Никакие данные не отправляются на сервера
- ❌ Нет телеметрии, аналитики, трекинга
- ❌ Нет рекламы

### Криптография

| Компонент | Алгоритм |
|-----------|----------|
| Обмен ключами | **X25519** (ECDH) |
| Шифрование данных | **AES-256-GCM** |
| Вывод ключей | **HKDF-SHA256** |
| Проверка кодов сопряжения | **HMAC-SHA256** (constant-time) |

### Защита от атак

- **MITM** — визуальное сравнение 6-значных кодов
- **Brute force** — блокировка на 30 сек после 3 неудачных попыток
- **Replay** — nonce в каждом сообщении
- **Tampering** — AES-GCM включает аутентификацию
